## 🏗️ AI System Design Thinking

When I build AI-powered backends, I think in pipelines — not just API calls.

```
User Query
    │
    ▼
[Query Pre-processing]  ── intent classification, query rewrite
    │
    ▼
[Retrieval Layer]  ──────  hybrid search (dense + sparse), FAISS / Pinecone
    │
    ▼
[Re-ranking]  ───────────  cross-encoder scoring, context compression
    │
    ▼
[LLM Generation]  ───────  GPT-4 / Mistral / LLaMA with structured prompting
    │
    ▼
[Post-processing]  ──────  output validation, citation grounding, streaming
    │
    ▼
[Async API Response]  ───  FastAPI + Redis queue + WebSocket for streaming
```

**Core design principles I follow:**
- 🔁 **Async-first** — Celery + Redis for background inference tasks, no blocking I/O
- 🧱 **Modular pipelines** — each stage (retrieval, rerank, generate) is independently swappable
- 📊 **Observability** — MLflow + W&B for tracking; structured logging for every inference call
- 💰 **Cost-aware** — prompt compression, caching (semantic + exact), model tiering by query complexity
- 🔐 **Production-ready** — rate limiting, input sanitization, output validation schemas
